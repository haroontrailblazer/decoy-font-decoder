The message is in the blur.

Decoy fonts hide two messages at different spatial frequencies. This guide explains the complete decoder: from Gaussian low-pass filtering to layer subtraction, contrast recovery, optional OCR, and agent verification.

01 · Signal model

One image, two spatial frequency bands.

A decoy-font glyph combines a sharp thin-outline letter with a broad blurred letter. Both occupy the same area, but they distribute ink very differently—and that difference makes the layers separable.

A · High frequency

The outline is the decoy

Thin, crisp strokes create strong local edges with very little total ink mass. OCR and naive visual reads lock onto those edges and confidently return the planted fake text.

B · Low frequency

The blur is the message

Wide, soft letter shapes spread more ink over a larger area. They survive aggressive Gaussian smoothing even after the thin outline strokes have effectively disappeared.

C · Perception

Scale decides the reading

At normal size the outline dominates. Squint, step back, or downsample and the sharp detail is suppressed while the broad hidden glyph becomes easier to see.

D · Separation

Both layers are recoverable

A low-pass filter recovers the hidden message. Subtracting that low-frequency result from the inverted input isolates the sharp decoy outlines as a second verifiable image.

Core signal I(x) = 255 − image(x) σ = 0.01 × max(width, height) hidden(x) = Gσ * I(x) decoy(x) = max(I(x) − hidden(x), 0)

Inversion turns ink mass into signal. The Gaussian low-pass retains the broad hidden letters while erasing thin outlines. Normalization and gamma make the hidden layer readable; the high-frequency remainder becomes the isolated decoy layer.

02 · Decoder pipeline

From one image to two readable layers.

The production decoder is deliberately classical: OpenCV and NumPy perform the recovery, while OCR is a final optional reader rather than the source of the signal.

01

Load the image in grayscale

OpenCV reads the input directly as an 8-bit grayscale image. Grayscale is sufficient because the separation depends on spatial frequency and ink mass, not on color.

If the file cannot be opened, the command stops with an explicit path error instead of producing misleading blank outputs.

02

Invert ink into signal

Subtracting the image from 255 turns dark strokes into high values. Both the soft hidden letters and the crisp outline strokes can then be filtered and subtracted numerically.

The array is converted to floating point before filtering so intermediate values keep their precision.

03

Scale sigma to the image

Sigma is the long edge multiplied by --sigma-frac. The 0.01 default adapts the blur radius to different source resolutions.

Resolution-aware scaling avoids a fixed pixel radius that works on one image size but fails on another.

04

Apply the Gaussian low-pass

A Gaussian blur with the computed sigma removes narrow, high-frequency outline strokes while preserving the wide low-frequency letter mass.

This is the computational equivalent of squinting or viewing the image from farther away.

05

Normalize the hidden mass

The low-pass result is stretched across the full 0–255 range. This makes faint but coherent blurred letters use the available contrast.

Normalization is deterministic and operates on the complete image rather than guessing individual character boundaries.

06

Gamma-boost the reveal

A default gamma of 0.6 strengthens the surviving soft shapes, then the result is inverted back to dark text on a white background.

The resulting revealed.png is the authoritative image for reading the real hidden message.

07

Subtract the hidden layer

The blurred low-frequency mass is subtracted from the inverted source. Negative values are clipped to zero, leaving the high-frequency remainder.

That remainder contains the thin outline letters used as the planted decoy text.

08

Normalize and write both files

The high-frequency remainder is normalized and inverted into dark-on-white decoy.png. The low-frequency result is written as revealed.png.

The output contract is always exactly two images so the agent can verify both readings without accumulating diagnostic variants.

09

Run optional OCR and verify

Tesseract receives revealed.png with page segmentation mode 6 and can provide a rough text hint when installed.

OCR is never treated as ground truth. The agent inspects both generated images, reports the real and fake texts, and marks any ambiguous glyph explicitly.

03 · Implementation details

The controls tune scale, not meaning.

Each value controls a clear stage of the frequency split. The defaults match the bundled examples and remain predictable across image resolutions.

Value Role Why it exists
sigma fraction = 0.01 Scale the Gaussian blur About one percent of the long edge removes thin outline strokes.
sigma fraction = 0.015 Only permitted retry Helps when the reveal still contains too much sharp outline detail.
gamma = 0.6 Boost revealed contrast Strengthens faint low-frequency mass without thresholding characters.
NORM_MINMAX Use the full tonal range Makes both separated layers visually inspectable.
float32 processing Preserve filter precision Avoids destructive clipping during blur and subtraction.
clip at zero Keep positive remainder Prevents the subtracted high-frequency layer from wrapping below black.
PSM 6 Optional OCR layout Treats the revealed result as one uniform block of text.
--no-ocr Disable text recognition Both visual outputs remain complete when Tesseract is unavailable.

Complexity

For an image of width W and height H, filtering and subtraction require approximately O(WH) work and O(WH) live memory for the grayscale, low-pass, and output arrays.

Trust boundary

The algorithm runs locally in the environment executing the skill. No model weights are needed for recovery, and the image is not required to leave that environment. OCR can be disabled entirely with --no-ocr.

04 · Outputs

Two images answer two different questions.

The revealed layer answers “what does the hidden message say?” The decoy layer answers “what sharp fake text was planted for the machine?”

revealed.png

Real hidden message

Dark, soft glyphs on white after low-pass filtering, normalization, and gamma boost. This is the authoritative layer to read.

decoy.png

Fake sharp message

The normalized high-frequency remainder left after subtracting the blurred mass. It records the trap the raw image encourages AI to read.

05 · CLI reference

Run the same decoder without an agent.

Python 3.8+, OpenCV, and NumPy are required. Tesseract and pytesseract are optional. The images are still generated when OCR is unavailable.

Install and decode 
pip install -r requirements.txt
python decode.py decoy-font-message.png
Tuned run 
python decode.py decoy-font-message.png \
  -o decoded \
  --sigma-frac 0.015 \
  --gamma 0.6 \
  --no-ocr
Argument Default Behavior
image required Path to a PNG, JPEG, WebP, or any image readable by the installed OpenCV build.
-o, --out-dir current directory Destination for revealed.png and decoy.png.
--sigma-frac 0.01 Sets Gaussian sigma as a fraction of the image's long edge.
--gamma 0.6 Controls contrast enhancement for the revealed low-frequency layer.
pytesseract optional Provides a rough OCR hint when the Tesseract executable is available.
--no-ocr false Skips Tesseract while preserving both visual outputs.

06 · Failure modes

What weak evidence means—and what to try next.

Sharp outlines remain

The blur radius is too small for the source resolution. Retry the same decoder once with --sigma-frac 0.015 so more thin outline detail is removed.

Hidden letters merge together

Sigma is too large or the original blurred glyphs overlap heavily. Return toward the 0.01 default so neighboring soft shapes remain distinguishable.

The reveal looks washed out

Adjust --gamma conservatively. Lower values strengthen faint mass; higher values preserve more tonal separation between adjacent shapes.

One character is missing

Inspect revealed.png before concluding the character is absent. Soft, rounded, blobby letters are the expected output—not a decoding defect.

OCR disagrees with the image

Trust visual evidence over OCR. Tesseract can confuse softened strokes. The agent workflow requires opening both output images, verifying both texts, and marking ambiguous characters.

Tesseract is unavailable

This is not a decoding failure. The computer-vision pipeline still writes both images. Read the revealed layer directly or install Tesseract later; use --no-ocr to skip the check explicitly.

08 · Security meaning

Obfuscation changes the sensor, not the information.

Decoy fonts can defeat tools that privilege crisp edges, but they do not remove the signal. They move the real message to a lower spatial frequency.

Technical verdict

If a human can reveal the words by squinting, a system that applies the equivalent low-pass filter can reveal them too.

Gaussian filtering, contrast normalization, and layer subtraction are established image-processing operations. The decoder does not break encryption because no encryption exists. It reconstructs a deliberately preserved visual signal from the viewing scale at which that signal was encoded.
